# Brainstorm Notes

> 最近更新：2026-06-05

这里保存产品脑暴和调研底稿。它回答“我们怎么想到当前方向”，但不单独作为实现或提交的权威来源。

## 当前入口

- `brainstorm.md`：用户收敛后的产品方向和数据结构构想。
- `aichats.md`：围绕剧本创作、中文剧本格式、Fountain、YAML、AST 和多格式导出的对话底稿。

## 已收敛结论

- 只生成 YAML 不够。作者需要能看懂、编辑和继续打磨剧本初稿。
- 工具应以“小说转结构化剧本”为主线，编辑器能力服务于生成后的可编辑性。
- 底层应采用结构化 AST / 项目模型，导出时生成 YAML。
- Fountain、Ren'Py、Naninovel、Word 等更适合作为后续导出目标，而不是 72 小时 MVP 的硬交付。
- 中文剧本需要考虑中文场景行、全角标点、角色名对白格式和影视/短剧/VN 的不同阅读习惯。

## 使用规则

- 这里可以保留未定想法，但进入实现前需要同步到 `../architecture/`、`../schema/` 或 `../../planning/`。
- 长对话中的技术建议需要经过提交需求审计，不能直接扩大 MVP 范围。
- 如果调研结论成为稳定架构，应迁入 `../architecture/` 并在本目录保留来源链接。
