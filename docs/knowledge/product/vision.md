# 月点产品愿景

> 最近更新：2026-06-05

月点的长期目标是成为一个 AI 辅助的语义剧本 IDE。

当前提交阶段只做题目要求内的 MVP：导入小说、理解章节、转换剧本 AST、提供基础语义块编辑、导出 YAML，并提交 YAML Schema 文档。长期 IDE 目标不进入当前 MVP 承诺，但它会指导我们今天的架构选择。

## 北极星

月点希望让故事创作者不用从空白剧本文档开始。

用户可以先导入小说文本，让 AI 生成一个结构化剧本初稿；也可以从一个灵感、主题、角色关系或世界观设定开始，让 AI 先生成故事种子，再继续发展成剧本 AST。随后用户在语义编辑器里改场景、改对白、补动作、调整节奏；最后按需要导出 YAML、Fountain-like 阅读稿、短剧剧本或视觉小说脚本片段。

因此月点不是单纯的 `novel -> yaml` 转换器。它的长期形态是：

> 以 AI 创作入口为起点，以 AST 语义编辑为核心，以多格式投影为出口的剧本 IDE。

## 当前 MVP 和长期 IDE 的边界

MVP 必须完成：

- 小说文本导入。
- 3 个章节以上样例的长文本处理能力。
- 章节解析和来源映射。
- 剧本 AST 生成。
- 基础语义块编辑。
- YAML 导出。
- YAML Schema 文档。
- demo 和 README。

长期 IDE 可以探索：

- 灵感生成模式：从主题、关键词、角色关系或世界观设定生成故事种子。
- 故事大纲模式：从短梗概扩写为章节、场景和人物弧线。
- 小说改编模式：从已有小说长文本改编为剧本。
- 场景卡片墙和结构大纲。
- 角色关系、出场统计和弧线追踪。
- 场景节奏、对白密度和时长估算。
- 多版本草稿和局部重生成。
- Fountain-like、短剧、视觉小说、制片表等多 projection。
- 更完整的剧本排版和阅读体验。
- 资产、设定、世界观和人物档案管理。

第一版不把长期 IDE 能力作为交付承诺。它们只能作为架构延展方向和 README 的未来展望。

## 产品层次

### 1. 创作入口与理解

月点首先要支持多种创作起点，而不是只处理短 prompt，也不是只处理小说导入。

当前提交只实现小说导入，因为官方题目要求证明“小说文本自动转换为剧本”的能力。长期产品可以接入 `ai-visual-novel-engine` 中类似灵感生成的模式，把主题、关键词、角色设定和世界观设定也当作 source。

关键能力：

- 粘贴或上传小说文本。
- 输入灵感、主题、角色关系或世界观设定。
- 选择 source type，例如 novel、inspiration seed、outline。
- 识别章节边界。
- 保留章节来源映射。
- 为后续模型生成提供分段上下文。

### 2. AI 生成与改编

AI 生成与改编的目标不是一段漂亮文本，而是可校验、可编辑、可导出的剧本 AST。

关键能力：

- 章节摘要。
- 角色候选。
- 场景拆分。
- 动作、对白、旁白、转场等语义块生成。
- diagnostics 反馈。

### 3. 语义编辑

月点的主编辑不是 YAML 编辑器，也不是 Fountain 文本编辑器。

主编辑对象是 AST 中的 scene、character 和 block。第一版只做轻量编辑；长期可以发展成完整剧本 IDE。

关键能力：

- 修改场景标题、地点、时间。
- 修改角色名、对白、动作和旁白。
- 增加、删除、调整语义块。
- 局部重生成。

### 4. 投影与导出

YAML 是官方提交格式，也是结构化结果的主要导出。其他格式是同一份 AST 的投影。

关键能力：

- YAML projection。
- Fountain-like preview。
- Ren'Py 或视觉小说片段 sample。
- 后续可能扩展到制片表、角色出场表和节奏报告。

### 5. 创作辅助

长期 IDE 的价值在于让作者继续工作，而不只是拿到一次生成结果。

未来可以支持：

- 角色弧线检查。
- 场景目的和冲突提示。
- 对白风格一致性检查。
- 场景节奏和时长估算。
- 版本对比和局部改写建议。

## 架构原则

长期 IDE 愿景会影响当前架构，但不能扩大 MVP 范围。

保留的原则：

- AST 是核心模型。
- YAML、Fountain-like、VN sample 都是 projection。
- 创作入口是 source adapter，小说导入只是当前 MVP 的 adapter。
- 编辑器编辑语义块，不强迫作者编辑 YAML。
- 校验层输出 diagnostics，不把问题藏在导出失败里。
- 小工具通过类型化输入输出组合成系统。

暂不实现的原则：

- 不做完整专业排版引擎。
- 不做完整 Fountain parser。
- 不做多人协作。
- 不做资产管理大系统。
- 不做完整 Story IDE。
